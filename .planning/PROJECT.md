# Ark — Project Definition

**Type:** Agentic Operating System (personal IP)
**Owner:** jongoldberg
**Repository:** https://github.com/goldiejz/ark

## Purpose

Ark is an autonomous orchestration substrate that compounds wisdom across projects. The user (CEO) sets vision; specialized agents (architect, engineers, QC, security, PM) execute autonomously; the system learns from each session and self-improves.

Named after the Ark of the Covenant — a portable sanctuary that preserves accumulated wisdom across the chaos of every project.

## Core Capabilities

1. **Project lifecycle automation** — create, align, deliver, verify, deploy, archive
2. **Pluggable employee registry** — agents defined as JSON, dispatched via subagent or CLI
3. **Self-improving observation** — continuous monitoring of system logs for patterns
4. **Cross-project learning** — Phase 6 daemon detects universal patterns
5. **Cost-aware dispatch** — 5-tier budget routing with auto-fallback
6. **Production safety** — gated deploys, verification loops, atomic commits

## Stakeholders

- **CEO** (user): vision, strategic decisions, production approvals
- **Agents**: design, engineering, QC, security, PM (all autonomous)
- **Vault**: persistent state across sessions and machines

## Out of Scope

- Multi-user/multi-tenant — personal IP, single owner
- Real-time collaboration — async via vault git commits
- IDE integration beyond Claude Code — CLI is the primary surface
