## Deviations from plan

### [Rule 1 — Bug] Cleanup trap dropped when `exec python3` replaced bash

The plan specified `exec python3 -m http.server …` for the foreground server. Smoke-testing surfaced two real defects:

1. `exec` replaces the bash process — the cleanup trap is lost; the regen-loop child becomes orphaned (PPID=1).
2. macOS `mktemp -d -t name` returns paths in `$TMPDIR` (not `/tmp`), so the leftover tmpdir wasn't visible to a `/tmp` glob — but the leak was real.

**Fix:** run python as a child (`python3 … &`), capture `$HTTP_PID`, and busy-wait via `kill -0 $HTTP_PID; sleep 1` in a polling loop with a `STOP=1` flag set by INT/TERM trap. Bash 3 `wait <pid>` is uninterruptible by signals, but polling `kill -0` with a 1s tick gives ≤1s cleanup latency. Trap unconditionally kills both children and `rm -rf` the tmpdir.

Smoke-test now confirms cleanup completes within ~1s of `kill -TERM` and within ~1s of natural python exit.

### [Rule 1 — Bug] `read -[pr]` constraint regex matched legitimate stream reads

The plan's smoke-test grep `grep -E 'read -[pr]' | grep -v 'AOS: intentional'` would match any `read -r` (stream-pipe parsing), not just the user-prompt `read -p`. Tier A `ark-dashboard.sh` itself uses `read -r` widely. Annotated each stream-`read -r` line with `# AOS: intentional` to satisfy the plan's literal grep while preserving the actual constraint (no `read -p` user prompts — `grep -E 'read -p'` returns 0 matches in code).
