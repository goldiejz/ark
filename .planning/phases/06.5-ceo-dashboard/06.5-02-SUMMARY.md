## Accomplishments

- `dashboard` case branch added to `scripts/ark` (immediately after `verify)`, alphabetically grouped with delivery/verification commands).
- Three flag modes wired:
  - **default** (no flag) → `exec bash $VAULT_PATH/scripts/ark-dashboard.sh "$@"` — Tier A.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing critical functionality] Added explicit `--help` / `-h` handling for the dashboard subcommand**
- **Found during:** Reading the plan's `<action>` block + smoke-test list
- **Issue:** The plan's smoke-test list required `bash scripts/ark dashboard --help` to "show docs for both modes." The original `<action>` only described two branches (`--tui` vs default passthrough), with no `--help` arm. Without explicit handling, `--help` would have been passed through to `ark-dashboard.sh`, which only documents Tier A — Tier B docs would be hidden.
- **Fix:** Added a `--help|-h)` case arm with an inline heredoc that documents both invocation modes, the Tier A passthrough flags, and the cargo build instruction.
- **Files modified:** scripts/ark (inside the new dashboard case arm)
- **Committed in:** c7b5aac

**2. [Rule 2 - Missing critical functionality] Added explicit unknown-flag rejection (`--bogus → exit 1`)**
- **Found during:** Smoke-test list item: "ark dashboard --bogus shows usage, exit 1"
- **Issue:** A pure passthrough would have forwarded `--bogus` to ark-dashboard.sh and let it decide. ark-dashboard.sh's CLI parser may or may not reject unknown flags consistently. The plan required rejection at the dispatcher level.
- **Fix:** Added a `--*)` case arm that whitelists `--no-color` and `--section` for passthrough and rejects everything else with a usage hint and exit 1.
- **Files modified:** scripts/ark (inside the new dashboard case arm)
- **Committed in:** c7b5aac

### Structural deviation (not auto-fix)

- The plan suggested a single `if [[ "${1:-}" == "--tui" ]]; then ... else ... fi` shape. I replaced it with a `case "${1:-}" in ... esac` shape so all four modes (--tui, --help, unknown --flag, default/passthrough) live in one structure. This matches the existing `promote-lessons` and `learn` patterns in scripts/ark and avoids nested `elif` chains. Behaviour for the two plan-defined modes (--tui and default) is identical.

**Total deviations:** 2 auto-fixed (Rule 2 — both required by the smoke-test contract but missing from the action steps). One structural shape change with no behaviour delta.
