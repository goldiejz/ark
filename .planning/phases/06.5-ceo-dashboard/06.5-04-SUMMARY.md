## App state shape

```rust
pub struct App {
    pub db: Db,
    pub portfolio_root: PathBuf,
    pub projects: Vec<Project>,
    pub recent_decisions: Vec<Decision>,
    pub class_counts: Vec<(String, i64)>,
    pub current_section: usize, // 0..NUM_SECTIONS
    pub should_quit: bool,
    pub last_tick: Instant,
}
```

`refresh()` re-runs project discovery + `recent_decisions(50)` + `counts_by_class()`. Errors swallowed → `Vec::default()`. Called once at construction and every 2s thereafter (when `last_tick.elapsed() >= 2000ms`).

## Section trait usage pattern (for 06.5-05 to copy)

```rust
pub struct PortfolioGrid;

impl Section for PortfolioGrid {
    fn name(&self) -> &'static str { "Portfolio" }

    fn render(&self, f: &mut Frame, area: Rect, app: &App) {
        let header = Row::new(vec!["Project", "Phase", "Last activity", "Health"])
            .style(Style::default().add_modifier(Modifier::BOLD));
        let rows: Vec<Row> = app.projects.iter().map(|p| { ... }).collect();
        let table = Table::new(rows, widths).header(header).block(...);
        f.render_widget(table, area);
    }
}
```

Each section is a unit struct (zero state). All state lives on `App`. `Section::render` is pure — given the same `&App`, produces the same widget tree.

## Decisions Made

See frontmatter `key-decisions`. Highlights:

1. **`--tui-no-alt-screen` skips raw mode entirely** — the plan's smoke test (`echo q | binary`) was uninvokable otherwise (raw mode requires TTY). Trade-off: the flag is now CI-only, not "alt-screen-off interactive".
2. **Depth-1 project discovery** — `~/code` is flat. Going deeper risks recursing into `node_modules`. Deepen later if a vault layout needs it.
3. **No `serde_json`** — hand-rolled `extract_int_field` keeps the dep tree at exactly 3 top-level crates. The two integer reads aren't worth 10+ transitive deps.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking issue] `--tui-no-alt-screen` couldn't be smoke-tested because raw mode requires a TTY**
- **Found during:** Task 2 verification (`echo q | binary --tui-no-alt-screen` failed with `Device not configured (os error 6)`)
- **Issue:** `enable_raw_mode()` fails when stdin isn't a TTY. The plan's verify command piped `echo q` into the binary — no TTY there.
- **Fix:** Made `--tui-no-alt-screen` skip BOTH raw mode AND alt screen (not just alt screen). Run loop in that mode renders one frame, reads stdin line-buffered, quits on 'q'. Documented in `--help` and SUMMARY.
- **Files modified:** `scripts/ark-dashboard/src/main.rs`
- **Net:** plan's verify path now works; interactive TUI still uses full raw+alt as designed.

**2. [Rule 2 - Missing critical functionality] Panic-safe terminal restore**
- **Found during:** Task 2 (plan said "Always restore terminal on panic — wrap the run loop body...")
- **Fix:** Added `install_panic_hook()` that wraps the original panic hook, calling `disable_raw_mode` + `LeaveAlternateScreen` + `DisableMouseCapture` before delegating to the original. Set up before `run_tui` is called.
- **Files modified:** `scripts/ark-dashboard/src/main.rs`
