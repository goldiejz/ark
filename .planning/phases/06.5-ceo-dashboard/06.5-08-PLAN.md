<objective>
Tier C of the CEO dashboard: a local web server serving a single-page HTML rendering of the same 7 sections as Tier A. Accessible from any browser on the local network (laptop, phone, second monitor).

**Why a third tier:** Tier A is terminal-only (CEO at the keyboard). Tier B (Rust TUI) is also terminal-only. Tier C makes the dashboard accessible from a phone or second screen — useful for "glance at progress while on a call" or "check from your phone in the kitchen."

**Architecture (deliberately simple):** No build step. No new language deps. Bash regenerates `index.html` from the same data Tier A reads. Python's stdlib `http.server` serves the static file. Browser auto-refreshes via `<meta http-equiv="refresh" content="5">`.

**Why this approach over Hono/Rust web server:**
- Zero new toolchain deps (python3 ships with macOS)
- No tokio/async runtime (Phase 6.5 explicitly excluded that for leanness)
- Same data source as Tier A (single rendering pipeline)
- Trivial to run; trivial to kill; trivial to debug
- Upgrade path: replace python http.server with a proper backend later (Phase 8) without changing the HTML
</objective>

    3. Background loop: re-run `regen_html()` every 5s. `(while true; do regen_html; sleep 5; done) &amp;` and trap to kill on EXIT.

    5. On startup, print a banner: `🌐 Ark Dashboard available at http://localhost:7919  (Ctrl-C to stop)`.
