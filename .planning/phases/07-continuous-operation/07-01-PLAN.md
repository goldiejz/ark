# inbox_parse_frontmatter <file_path>
#   Reads stdin/file with YAML-ish header (--- delimited).
#   Echoes lines:  KEY=value
#   Recognised keys: intent, customer, priority
#   Unknown keys: ignored. Missing required intent: returns 2.
#   Body extraction: sets DESC=<first non-blank line of body, no leading '# '>.

#   Echoes the command-string the daemon will eval, e.g.:
#     ark create "<desc>" --customer "<cust>"
#     ark deliver --phase N    # for new-phase (N from extra)
#     ark deliver               # for resume
#     ark promote-lessons       # for promote-lessons
#   Returns 0 on success; 1 on unknown intent.

# inbox_self_test
#   Runs 16+ assertions against an isolated mktemp -d fixture. Returns 0/1.

    Create `scripts/lib/inbox-parser.sh` (executable, `#!/usr/bin/env bash`, `set -uo pipefail`
    inside functions only — never `set -e` at file scope; lib must be safely sourceable).

    Functions to implement:

    1. `inbox_parse_frontmatter <file>` — uses awk to extract lines between first `---`
       and second `---`, then `key:value` split. For each recognised key
       (`intent|customer|priority`), echo `KEY=<value>` (UPPERCASE key, value
       trimmed). After frontmatter, find first non-blank body line; if it starts
       with `# `, strip the leading `# `; echo `DESC=<that line>`. Returns 2 if
       no `intent` key found OR no `---` delimiters.

    2. `inbox_validate_intent <intent>` — case statement against
       `new-project|new-phase|resume|promote-lessons`. Returns 0 or 1.

    3. `inbox_dispatch_intent <intent> <customer> <priority> <description> [extra]` —
       case statement; for each intent, printf the command string with proper
       single-arg quoting. Use printf '%q' for safety on description. For
       new-phase, `extra` is the phase number; for promote-lessons, no extra
       used. Returns 0 success / 1 unknown.

    4. `inbox_self_test` — creates `mktemp -d` fixture; writes 5 sample INBOX
       files (good, missing-intent, no-frontmatter, unknown-intent, full-fields);
       calls each function; counts PASS/FAIL with bash 3 arrays of strings (no
       declare -A); cleans up via trap RETURN; echoes `RESULT: N/16 pass` and
       returns 0 only if all pass.

    5. CLI guard at file end: `if [[ "${BASH_SOURCE[0]}" == "${0}" ]]; then` —
       handle `--self-test` arg by calling `inbox_self_test`. Default: silent
       no-op (lib sourcing produces no output).

    Bash 3 compat: no `declare -A`, no `mapfile`, no `${var,,}`. Use `tr` for case
    folds, `awk` for parsing, `sed` for trim. NO `read -p` anywhere.

    Per CONTEXT.md D-CONT-INTENTS: dispatch table is verbatim from the locked

    Per CONTEXT.md D-CONT-INBOX-FORMAT: defaults customer=scratch, priority=medium
