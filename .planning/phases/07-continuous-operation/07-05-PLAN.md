    1. Add a `cmd_continuous` function:
       ```bash
       cmd_continuous() {
         local sub="${1:-}"
         shift || true
         case "$sub" in
           install)   bash "$VAULT_PATH/scripts/ark-continuous.sh" --install "$@" ;;
           uninstall) bash "$VAULT_PATH/scripts/ark-continuous.sh" --uninstall "$@" ;;
           status)    bash "$VAULT_PATH/scripts/ark-continuous.sh" --status "$@" ;;
           pause)     bash "$VAULT_PATH/scripts/ark-continuous.sh" --pause "$@" ;;
           resume)    bash "$VAULT_PATH/scripts/ark-continuous.sh" --resume "$@" ;;
           tick)      bash "$VAULT_PATH/scripts/ark-continuous.sh" --tick "$@" ;;
           ""|help|-h|--help)
             cat <<EOF >&2
       usage: ark continuous <subcommand>
         install     generate + load ~/Library/LaunchAgents/com.ark.continuous.plist
         uninstall   unload + remove the plist
         status      show last tick / next tick / recent decisions / daily tokens
         pause       create PAUSE file (kill-switch)
         resume      remove PAUSE file
         tick        run one tick manually (for debugging)
       EOF
             return 2
             ;;
           *)
             echo "unknown subcommand: $sub" >&2
             return 2
             ;;
         esac
       }
       ```

    2. Add the case arm in the main dispatcher:
       ```bash
       continuous) cmd_continuous "${@:2}" ;;
       ```

    3. Update the dispatcher's help text / default usage block to include
       `continuous` alongside `dashboard`, `deliver`, etc.

    4. Verify backward-compat: do not modify any other `cmd_*` function or
       case arm. Diff should show only additions.

      ark-continuous.sh.
    - No `read -p`. No `set -e` at function level.
    - Exit codes propagate: subshell's exit code is the dispatcher's exit code
      (Bash default behavior; do not wrap with `|| true`).
    - "$@" forwarding preserves quoting for any future flags.
  </action>
