<objective>
Extend `scripts/ark-continuous.sh` (inside the `# === SECTION: subcommands
(Plan 07-04) ===` sentinel laid by 07-02) with five subcommand implementations
+ a launchd plist generator:

1. `continuous_install` — generate plist at
   `${ARK_LAUNCHAGENTS_DIR:-$HOME/Library/LaunchAgents}/com.ark.continuous.plist`
   with `StartInterval = tick_interval_min * 60` (read from policy.yml, default
   15 → 900s). Atomic tmp+mv. Best-effort `launchctl load` (warns on failure
   instead of erroring). Idempotent: regenerate is byte-identical when policy
   unchanged.

2. `continuous_uninstall` — best-effort `launchctl unload`; `rm -f` plist.

3. `continuous_status` — prints last tick timestamp, next tick estimate
   (last + StartInterval), last 10 `class='continuous'` audit rows, today's
   token usage / daily cap, PAUSE state. Read-only.

4. `continuous_pause` — `touch ~/vaults/ark/PAUSE`; log `PAUSE_ACTIVE`
   (manual class indicator). Idempotent.

5. `continuous_resume` — `rm -f ~/vaults/ark/PAUSE`; log resume audit row.
   Idempotent.

CLI guard at file end is extended to dispatch `--install / --uninstall /
--status / --pause / --resume / --tick / --self-test`.

Purpose: Phase 7 acceptance criteria #3, #4, #5. The launchd plist is the
delivery mechanism for "drop intent, walk away, return to find shipped" demo.

Output: Extended `scripts/ark-continuous.sh` with subcommands populated +
self-test additions for plist generation (against ARK_LAUNCHAGENTS_DIR=tmp).
</objective>
