<objective>
Build the main continuous-operation daemon body. Sources the Plan 07-01 parser,
sources ark-policy.sh for `_policy_log` (single-writer audit), and implements
the core tick loop:

1. Acquire mkdir-lock; defer if contended (LOCK_CONTENDED audit).
2. Check PAUSE file; exit no-op if present (PAUSE_ACTIVE audit).
3. Check daily token cap; exit SUSPENDED if exceeded (DAILY_CAP_HIT audit).
4. Scan `~/vaults/ark/INBOX/*.md`; for each:
   a. Parse frontmatter via inbox_parse_frontmatter.
   b. If malformed → rename `.malformed`, log INBOX_MALFORMED.
   c. Else build dispatch command via inbox_dispatch_intent + eval it under
      a sub-shell with timeout.
   d. On success → mv to `processed/<UTC-date>/`, log INBOX_PROCESSED.
   e. On failure → rename `.failed`, append ESCALATIONS entry, log INBOX_FAILED.
5. Release lock (trap EXIT).
6. Log TICK_COMPLETE with summary counts.

Lays sentinel comment sections for 07-03 (health-monitor) and 07-04
(subcommands) so they extend the same file without merge conflicts.

Purpose: This is the irreducible core of Phase 7 — INBOX in, processed-or-failed
out, every decision audit-logged, no `read -p`, no silent drops. Phase 7's
"intent → ship" demo from CONTEXT.md hinges on this loop being correct.

Output: `scripts/ark-continuous.sh` with tick + process_inbox + lock + daily-cap
helpers + sentinel sections + self-test against synthetic INBOX.
</objective>

         policy.yml `continuous.dispatch_timeout_sec`). On exit 0 → ensure
         `processed/$(date -u +%Y-%m-%d)` exists; `mv` file there;
         `_policy_log "continuous" "INBOX_PROCESSED" "intent:$INTENT"

         return 0.
       - On non-zero exit → rename `${file%.md}.failed`; append to
         `~/vaults/ark/ESCALATIONS.md` (single-writer route via existing helper
         if available; else direct atomic append with markdown entry containing
         file path + intent + exit code); `_policy_log "continuous"
         "INBOX_FAILED" "exit:$rc" "{\"file\":\"$(basename "$file")\"}"`; return 1.

         return 0.
       - If [[ -f "$PAUSE_FILE" ]] → `_policy_log "continuous" "PAUSE_ACTIVE" ...`;
