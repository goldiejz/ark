  <action>
    Append a new section after the Phase 6.5 dashboard contract:

    ```markdown
    ## AOS Continuous Operation Contract (Phase 7)

    ### Surfaces
    - `scripts/ark-continuous.sh` — daemon (tick + INBOX lifecycle + lock + cap + health monitor + subcommands).
    - `scripts/lib/inbox-parser.sh` — frontmatter parser + intent dispatcher.
    - `scripts/ark-weekly-digest.sh` — weekly aggregator + standalone plist.
    - `scripts/ark continuous <subcmd>` — user-facing dispatcher arm.

    ### INBOX format
    - Files at `~/vaults/ark/INBOX/*.md`.
    - YAML-ish frontmatter: `intent` (required), `customer` (default `scratch`),
      `priority` (default `medium`), `phase` (only for new-phase intent).
    - Body: first non-blank line (with leading `# ` stripped) used as the
      description.

    ### Intents
    - `new-project` → `ark create "<desc>" --customer "<customer>"`
    - `new-phase` → `ark deliver --phase <N>`
    - `resume` → `ark deliver` (portfolio engine picks)
    - `promote-lessons` → `ark promote-lessons`

    ### Lifecycle
    - Success → `mv` to `INBOX/processed/<UTC-date>/`.
    - Failure → rename `<file>.failed` + ESCALATIONS.md entry.
    - Parse failure → rename `<file>.malformed` + audit row.
    - Files NEVER silently dropped.

    ### Safety rails
    - mkdir-lock at `~/vaults/ark/.continuous.lock` prevents tick overlap.
    - PAUSE file at `~/vaults/ark/PAUSE` halts all ticks (kill-switch).
    - Daily token cap (`policy.yml::continuous.daily_token_cap`, default 50000)
      → SUSPENDED until UTC date rollover.
    - Auto-pause: 3 consecutive failure-ticks → PAUSE file created + escalate.

    ### Health monitor
    - Stuck-phase signal: STATE.md mtime > 24h AND no commits in 24h.
    - 3 consecutive detections within 60min on same (project, phase) →
      one ESCALATIONS entry (24h dedupe).

    ### Audit class
    - All decisions logged via `_policy_log "continuous" "<DECISION>" ...`.
    - Decisions: TICK_START, TICK_COMPLETE, INBOX_PROCESSED, INBOX_FAILED,
      INBOX_MALFORMED, STUCK_PHASE_DETECTED, STUCK_ESCALATED, DAILY_CAP_HIT,
      AUTO_PAUSED, LOCK_CONTENDED, PAUSE_ACTIVE, WEEKLY_DIGEST_WRITTEN.

    ### launchd plists (macOS)
    - `~/Library/LaunchAgents/com.ark.continuous.plist` — main daemon,
      StartInterval = `tick_interval_min * 60` (default 900s).
    - `~/Library/LaunchAgents/com.ark.weekly-digest.plist` — separate weekly
      job, StartCalendarInterval Weekday=0 Hour=9 Minute=0.

    ### Read-only invariants
    - Daemon writes only to: `INBOX/`, `INBOX/processed/`, `ESCALATIONS.md`,
      `PAUSE`, `.continuous.lock`, `observability/policy.db` (via
      `_policy_log` single-writer), `observability/continuous-operation.log`,
      `observability/weekly-digest-YYYY-WW.md`.
    - Daemon NEVER writes outside `~/vaults/ark/`.
    ```

    If `STRUCTURE.md` does not exist locally, write to vault equivalent (best-
    effort path: `~/vaults/ark/STRUCTURE.md`); record path in SUMMARY.
