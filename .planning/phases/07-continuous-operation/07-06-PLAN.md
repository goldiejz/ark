<objective>
Build a standalone weekly digest generator that aggregates the previous 7 days
of `policy.db` activity into a markdown report. Runs from a separate launchd
plist (Sunday 09:00 local) so it doesn't compete with the main daemon's
StartInterval cadence.

Six sections per CONTEXT.md D-CONT-WEEKLY-DIGEST:
1. Projects shipped — distinct projects with `class='dispatcher'`
   `decision='ROUTED'` rows where outcome=success in the week.
2. Phases completed — distinct (project, phase) pairs from STATE.md mtime
   diff (or audit-log signal).
3. Escalations resolved — count of `class='escalation'` `decision='RESOLVED'`
   rows in the week (or count from ESCALATIONS.md status: resolved entries).
4. Learner promotions — `class='self_improve'` PROMOTED rows in the week
   + `class='lesson_promote'` PROMOTED rows.
5. Budget burn — sum of `json_extract(context, '$.tokens')` from
   `class='budget'` rows.
6. Dashboard URL — static line: `Run \`ark dashboard\` (or
   \`ark dashboard --web\`) for live view`.

Output filename: `weekly-digest-YYYY-WW.md` where `WW` is ISO week (`date +%V`
on macOS BSD; macOS supports `%V` since 10.5).

Purpose: Phase 7 acceptance criterion #7. Closes the "user is async, what
happened this week?" loop.

Output: `scripts/ark-weekly-digest.sh` with `weekly_digest_generate`,
`weekly_digest_install` (separate plist), self-test, CLI guard.
</objective>

# weekly_digest_generate [--week YYYY-WW]
#   Aggregates last 7 days of policy.db (or ISO week if --week given) into

    Functions:

    1. `weekly_digest_generate [--week YYYY-WW]`:
       - Determine week: default `$(date +%G-%V)` (ISO year + week, BSD macOS
         supports both since 10.5). Compute `since_iso` = `$(date -v -7d +%FT%T)`
         (BSD `date -v`).
       - out="$OBSERVABILITY/weekly-digest-$week.md"
       - tmp="$(mktemp)"
       - Build markdown: `# Ark Weekly Digest — Week $week` then 6 sections.
         Each section uses `sqlite3 -readonly` queries against policy.db.
         Use `IFNULL(...,0)` for empty results to keep table cells populated.
       - Section 1 (Projects shipped): `SELECT DISTINCT json_extract(context,'$.project') FROM decisions WHERE class='dispatcher' AND decision='ROUTED' AND outcome='success' AND ts >= '$since_iso'` → bullet list.
       - Section 2 (Phases completed): `SELECT json_extract(context,'$.project') AS p, json_extract(context,'$.phase') AS ph, COUNT(*) FROM decisions WHERE class='dispatcher' AND decision='ROUTED' AND ts >= '$since_iso' GROUP BY p, ph` → table.
       - Section 3 (Escalations resolved): `SELECT COUNT(*) FROM decisions WHERE class='escalation' AND decision='RESOLVED' AND ts >= '$since_iso'`.
       - Section 4 (Learner promotions): two queries (self_improve PROMOTED + lesson_promote PROMOTED) → bullet list of decision_ids + reasons.
       - Section 5 (Budget burn): `SELECT COALESCE(SUM(CAST(json_extract(context,'$.tokens') AS INTEGER)),0) FROM decisions WHERE class='budget' AND ts >= '$since_iso'`.
       - Section 6 (Dashboard URL): static line.
       - `mv "$tmp" "$out"` (atomic on same FS — both under $VAULT_PATH).
       - `_policy_log "continuous" "WEEKLY_DIGEST_WRITTEN" "week:$week" "{\"file\":\"$(basename "$out")\"}"`

    2. `weekly_digest_install`:

       - if [[ -z "${ARK_LAUNCHAGENTS_DIR:-}" ]] && command -v launchctl;
