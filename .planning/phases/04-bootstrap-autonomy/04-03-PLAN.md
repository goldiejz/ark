<interfaces>
The base template uses `{{ANCHOR}}` placeholders. 04-04's substitution will use:
```bash
sed -e "s|{{PROJECT_NAME}}|$NAME|g" \
    -e "s|{{PROJECT_TYPE}}|$TYPE|g" \
    -e "s|{{CUSTOMER}}|$CUSTOMER|g" \
    -e "s|{{CREATED_DATE}}|$DATE|g" \
    -e "/{{ADDENDUM}}/r $ADDENDUM_FILE" -e "/{{ADDENDUM}}/d" \
    -e "/{{CUSTOMER_FOOTER}}/r $FOOTER_FILE" -e "/{{CUSTOMER_FOOTER}}/d" \
    bootstrap/claude-md-template.md > "$PROJECT_DIR/CLAUDE.md.tmp"
mv "$PROJECT_DIR/CLAUDE.md.tmp" "$PROJECT_DIR/CLAUDE.md"
```

Anchors MUST be on their own line (for the `r` + `d` pattern to work). String anchors (`{{PROJECT_NAME}}`, `{{CUSTOMER}}`, `{{PROJECT_TYPE}}`, `{{CREATED_DATE}}`) can appear anywhere.
</interfaces>
