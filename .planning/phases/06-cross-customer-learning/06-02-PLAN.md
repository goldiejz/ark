    5. **Public: `promoter_scan_lessons [root]`**:

         - title = output of `lesson_extract_body | head -1` (the first non-blank

    6. **Public: `promoter_cluster_similar`** (reads scan TSV from stdin):
       Greedy single-link clustering:
       - Maintain an array of `seeds[i]=tmp_path` (Bash 3: indexed array, not
         associative). Threshold = 60.
       - For each input row, walk seeds; first seed whose
         `lesson_similarity $row_path $seeds[i]` ≥ 60 → assign cluster_id=i,
         emit row with similarity score.
       - No match → push as new seed; emit row with similarity=100 (self-seed).
       - Emit TSV: `cluster_id<TAB>customer<TAB>lesson_path<TAB>title<TAB>severity<TAB>similarity_to_seed`.
       - Pure Bash + awk. Quadratic in #lessons (acceptable — total lesson
         volume is < 200 across 3 real customers; Tier 12 fixture is < 30).

    7. **Public: `promoter_classify_cluster`** (reads cluster TSV from stdin):
       For each cluster_id:
       - `customer_count = unique customers in cluster`
       - `lesson_count = total rows`
       - severity = `anti` if ANY row in cluster has severity=anti, else max
       - title_seed = title of the row with smallest cluster_id index
       - **Conflict heuristic** (D-NO-DEPRECATE-CUSTOMER + risk #3 from
         CONTEXT.md): if titles within the cluster contain mutually-contradicting
         verbs (any title contains literal `do ` AND any other contains
         literal `don't`/`do not`, on the SAME tokenised noun ≥1 overlap),
         emit verdict = DEPRECATED with route=none. (This is intentionally
         narrow — it's a guard, not the full conflict-resolution feature
         which CONTEXT.md explicitly defers.)
       - Else if customer_count ≥ 2 AND lesson_count ≥ 3 → verdict = PROMOTE.
         Route = `anti-patterns` if severity=anti, else `universal-patterns`.
       - Else → verdict = MEDIOCRE_KEPT_PER_CUSTOMER. Route = none.
       - Emit TSV: `cluster_id<TAB>verdict<TAB>customer_count<TAB>lesson_count<TAB>route<TAB>title_seed`.

    8. **Public: `promoter_run [--full | --since DATE] [--apply] [--dry-run]`**:
       Orchestrator. Constructs scan → cluster → classify pipeline:
       ```
       trap 'rm -rf "$scan_tmpdir"' EXIT
       scan_tsv=$(promoter_scan_lessons "$ARK_PORTFOLIO_ROOT")
       cluster_tsv=$(echo "$scan_tsv" | promoter_cluster_similar)
       verdicts_tsv=$(echo "$cluster_tsv" | promoter_classify_cluster)
       # summary
       p=$(echo "$verdicts_tsv" | awk -F'\t' '$2=="PROMOTE"' | wc -l)
       d=$(echo "$verdicts_tsv" | awk -F'\t' '$2=="DEPRECATED"' | wc -l)
       m=$(echo "$verdicts_tsv" | awk -F'\t' '$2=="MEDIOCRE_KEPT_PER_CUSTOMER"' | wc -l)
       echo "clusters: $((p+d+m)) (promote: $p, deprecate: $d, mediocre: $m)"
       ```
       `--since` parses the date and filters scan rows by lesson-file mtime
       (single `find -newermt` call — BSD `find` supports it, with GNU fallback).
       `--apply` is the apply-pending hook; in 06-02 it just sets a flag and
       calls `promoter_apply_pending` if defined (Plan 06-03 defines it).
       Otherwise echoes `# Plan 06-03 has not been applied yet` and returns 0.
       `--dry-run` prints the verdicts TSV to stdout and skips apply entirely.
